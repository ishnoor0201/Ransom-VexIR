# Machine Learning Model Training Pipeline

## Documentation & Detailed Explanation

---

## 1. Overview

This script implements a **complete Machine Learning training and evaluation pipeline** using a normalized dataset. It performs:

* Data loading and preprocessing
* Feature normalization and scaling
* Training of multiple Machine Learning and Deep Learning models
* Performance evaluation using multiple metrics
* Model comparison and result summarization

The pipeline is designed for **binary classification problems** where the dataset contains numerical features and a target column named `label`.

---

## 2. Objectives of the Script

The main goals are:

1. Normalize dataset features
2. Train multiple models under identical conditions
3. Evaluate models using consistent metrics
4. Compare performance fairly
5. Automatically generate experiment results

---

## 3. Models Implemented

The script trains six different models:

### Classical Machine Learning Models

* XGBoost Classifier
* CatBoost Classifier
* Random Forest Classifier

### Ensemble Models

* Voting Ensemble
* Bagging Ensemble

### Deep Learning Model

* 1D Convolutional Neural Network (CNN)

---

## 4. Evaluation Metrics

Each model is evaluated using:

| Metric    | Purpose                            |
| --------- | ---------------------------------- |
| Accuracy  | Overall correctness                |
| Precision | Correct positive predictions       |
| Recall    | Ability to detect positives        |
| F1 Score  | Balance of Precision & Recall      |
| AUC       | Quality of probability predictions |

## 6. Data Preprocessing

### 6.1 Dataset Loading

The dataset is loaded from:

```
static_embeddings.csv
```

Expected structure:

| file | feature1 | feature2 | ... | label |
| ---- | -------- | -------- | --- | ----- |

* `file` → identifier (removed)
* `label` → target variable

---

### 6.2 Feature and Target Separation

```
X → Features
y → Target labels
```

---

### 6.3 Feature Normalization

Two scaling steps are applied:

#### Step 1 — MinMax Normalization

```
MinMaxScaler()
```

Transforms values into range:

```
[0, 1]
```

Purpose:

* Equal feature ranges
* Stable neural network training

---

#### Step 2 — Standard Scaling

```
StandardScaler()
```

Transforms data to:

```
mean = 0
standard deviation = 1
```

Purpose:

* Improves optimization
* Stabilizes gradients

---

### 6.4 Label Encoding

Categorical labels are converted to numeric values:

Example:

```
Fake → 0
Real → 1
```

---

### 6.5 Data Splitting

The dataset is split using:

```
train_test_split()
```

Configuration:

* Training data: **80%**
* Testing data: **20%**
* Random seed: **42**
* Stratified split: **Enabled**

#### Stratified Splitting

Maintains identical class proportions in both training and testing datasets.

This prevents biased evaluation.

---

## 7. Model Training

Each model follows the same workflow:

```
Initialize Model
        ↓
Fit Training Data
        ↓
Predict Test Data
        ↓
Calculate Probabilities
        ↓
Compute Evaluation Metrics
```

---

### 7.1 XGBoost

Gradient boosting decision trees.

Characteristics:

* High accuracy on tabular data
* Handles nonlinear relationships
* Robust to feature interactions

---

### 7.2 CatBoost

Gradient boosting algorithm optimized for stability.

Advantages:

* Reduced overfitting
* Strong default performance
* Efficient learning process

---

### 7.3 Random Forest

Ensemble of decision trees.

Concept:

* Multiple trees trained independently
* Final prediction via majority voting

Benefits:

* Low variance
* Good baseline model

---

### 7.4 Voting Ensemble

Combines predictions from:

* XGBoost
* Random Forest
* CatBoost

Uses **soft voting**:

```
Average predicted probabilities
```

Improves robustness.

---

### 7.5 Bagging Ensemble

Uses bootstrap aggregation:

* Trains multiple Random Forest models
* Each model sees random subsets of data

Goal:
Reduce overfitting and variance.

---

## 8. Deep Learning Model — CNN

### 8.1 Why CNN for Tabular Data?

The dataset contains embeddings or structured numerical features.
CNN learns **local feature interactions** between neighboring features.

---

### 8.2 CNN Architecture

#### Input Transformation

```
(batch, features)
→
(batch, 1, features)
```

---

#### Convolution Blocks

```
Conv1 → BatchNorm → ReLU → MaxPool
Conv2 → BatchNorm → ReLU → MaxPool
Conv3 → BatchNorm → ReLU
```

Purpose:

* Extract hierarchical feature patterns.

---

#### Fully Connected Layers

```
Flatten
→ Dense(128)
→ Dense(64)
→ Output(1)
```

Final activation:

```
Sigmoid
```

Outputs probability of class 1.

---

### 8.3 Training Strategy

* Optimizer: Adam
* Loss Function: Binary Cross Entropy
* Batch Size: 32
* Epochs: 100
* Dropout: 0.3 (regularization)

#### Early Stopping

Training stops if loss does not improve for 10 epochs.
